# Content Guidelines

This document provides guidelines for creating and managing content on the Betirement website.

## Overview

Content is the heart of the Betirement brand. These guidelines ensure consistency, quality, and effectiveness across all content types.

## Brand Voice and Tone

### Voice Characteristics

**Authentic**: Real experience, honest insights, genuine advice
- Share personal stories and lessons learned
- Admit mistakes and challenges
- Avoid hype and unrealistic promises

**Knowledgeable**: Expert guidance without being condescending
- Explain complex concepts clearly
- Provide actionable advice
- Back claims with data and experience

**Approachable**: Friendly and conversational, not academic
- Use "you" and "I" language
- Write like you're talking to a friend
- Avoid jargon unless necessary (then explain it)

**Empowering**: Help readers take control of their financial future
- Focus on what they can do
- Provide clear next steps
- Celebrate progress and wins

### Tone Guidelines

**Do**:
- ‚úÖ Be conversational and relatable
- ‚úÖ Use active voice
- ‚úÖ Keep sentences concise
- ‚úÖ Break up long paragraphs
- ‚úÖ Use examples and stories
- ‚úÖ Address reader directly ("you")

**Don't**:
- ‚ùå Use financial jargon without explanation
- ‚ùå Make guarantees about returns
- ‚ùå Be overly formal or academic
- ‚ùå Use fear-based marketing
- ‚ùå Criticize other approaches harshly
- ‚ùå Make it about you (focus on reader)

## Content Types

### Blog Posts

**Purpose**: Educate, inspire, and build trust

**Length**: 1,000-2,500 words (optimal for SEO and engagement)

**Structure**:
1. **Hook** (first 100 words): Grab attention, state the problem
2. **Introduction**: Set context, promise value
3. **Body**: Main content with subheadings
4. **Conclusion**: Summarize, provide next steps
5. **Call-to-Action**: Guide reader to next action

**Best Practices**:
- Start with a compelling headline (50-60 characters)
- Use subheadings every 200-300 words
- Include images every 300-500 words
- Add bullet points and numbered lists
- Include internal links to related content
- End with a clear CTA

**Example Structure**:
```markdown
# How to Calculate Your Bitcoin Retirement Number

*Discover the exact formula I used to retire at 51 with Bitcoin*

[Hook paragraph about retirement uncertainty]

[Personal story with numbers]

## Your Next Steps

[Action items for reader]

**Ready to calculate your number?** Download our free Bitcoin Retirement Calculator.
```

## SEO Guidelines

### Keywords

**Primary Keywords** (high volume, high intent):
- Bitcoin retirement
- Early retirement with Bitcoin
- Bitcoin retirement planning
- Retire on Bitcoin

**Secondary Keywords** (supporting topics):
- Bitcoin investment strategy
- Cryptocurrency retirement
- Financial independence Bitcoin
- Bitcoin portfolio allocation

**Long-Tail Keywords** (specific, lower competition):
- How to retire early with Bitcoin
- Bitcoin retirement calculator
- Is Bitcoin good for retirement
- Bitcoin retirement success stories

### Keyword Usage

**Title**: Include primary keyword near the beginning
- ‚úÖ "Bitcoin Retirement Planning: Complete Guide for 2024"
- ‚ùå "A Complete Guide to Planning Your Retirement with Bitcoin in 2024"

**URL**: Use keyword-rich, readable slugs
- ‚úÖ `/blog/bitcoin-retirement-planning-guide`
- ‚ùå `/blog/post-12345`

**Headings**: Include keywords naturally in H2 and H3
- ‚úÖ "How to Start Bitcoin Retirement Planning"
- ‚ùå "Getting Started"

**Body**: Use keywords naturally (1-2% density)
- Don't force keywords
- Use variations and synonyms
- Focus on readability first

**Meta Description**: Include primary keyword
- 150-160 characters
- Compelling call-to-action
- Unique for each page

### Internal Linking

**Best Practices**:
- Link to related content
- Use descriptive anchor text
- Link from high-traffic to low-traffic pages
- Create content clusters around topics

**Example**:
```markdown
If you're new to Bitcoin, start with our [Bitcoin Fundamentals Guide](/content/blog/bitcoin-basics) before diving into retirement planning.

For a deeper dive into portfolio allocation, check out our [Bitcoin Allocation Calculator](/resources/calculator).
```

## Image Guidelines

### Requirements

**Blog Post Cover Images**:
- Dimensions: 1200x630px (optimal for social sharing)
- Format: WebP (with JPG fallback)
- File size: < 200KB
- Alt text: Descriptive, includes keyword

**In-Content Images**:
- Dimensions: 800-1200px wide
- Format: WebP
- File size: < 150KB
- Alt text: Descriptive

**Thumbnails**:
- Dimensions: 400x300px
- Format: WebP
- File size: < 50KB

### Image Sources

**Preferred**:
- Original photography
- Custom graphics
- Licensed stock photos (Unsplash, Pexels)

**Avoid**:
- Generic stock photos
- Watermarked images
- Low-resolution images
- Copyrighted material

### Alt Text

**Good Alt Text**:
- ‚úÖ "Bitcoin price chart showing growth from 2017 to 2024"
- ‚úÖ "Retired couple reviewing their Bitcoin portfolio on laptop"
- ‚úÖ "Infographic showing Bitcoin retirement planning steps"

**Bad Alt Text**:
- ‚ùå "Image"
- ‚ùå "Bitcoin"
- ‚ùå "Click here"

## Formatting Standards

### Headings

Use proper heading hierarchy:

```markdown
# H1: Page Title (only one per page)
## H2: Main Sections
### H3: Subsections
#### H4: Minor Points (use sparingly)
```

### Lists

**Bullet Points** (unordered):
- Use for non-sequential items
- Keep items parallel in structure
- Start each item with capital letter
- No period if single sentence

**Numbered Lists** (ordered):
1. Use for sequential steps
2. Use for ranked items
3. Use for processes

### Emphasis

**Bold**: Important terms, key takeaways
**Italic**: Emphasis, book titles, foreign words
~~Strikethrough~~: Corrections, outdated info

### Links

**Internal Links**:
- Use descriptive anchor text
- Open in same tab
- Example: `[Bitcoin retirement calculator](/resources/calculator)`

**External Links**:
- Use descriptive anchor text
- Open in new tab (when appropriate)
- Add `rel="noopener noreferrer"` for security
- Example: `[YouTube channel](https://youtube.com/@betirement)`

### Code and Technical Content

Use code blocks for:
- Formulas
- Technical instructions
- Examples

```

$50,000 √ó 25 = $1,250,000
```

### Publishing Frequency

**Blog Posts**: 2-3 per week
- Monday: Educational content
- Wednesday: Strategy/How-to
